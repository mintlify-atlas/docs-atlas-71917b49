## Usage Example

<Tabs>
  <Tab title="Python">
    ```python
    import opencc
    
    # Initialize converter
    converter = opencc.OpenCC('s2t.json')
    
    # Convert Simplified to Traditional
    result = converter.convert('汉字')
    print(result)  # 漢字
    ```
  </Tab>
  <Tab title="Node.js">
    ```javascript
    const OpenCC = require('opencc');
    
    // Initialize converter
    const converter = new OpenCC('s2t.json');
    
    // Convert Simplified to Traditional
    converter.convertPromise("汉字").then(converted => {
      console.log(converted);  // 漢字
    });
    ```
  </Tab>
  <Tab title="C++">
    ```cpp
    #include "opencc.h"
    
    int main() {
      // Initialize converter
      const opencc::SimpleConverter converter("s2t.json");
      
      // Convert Simplified to Traditional
      std::string result = converter.Convert("汉字");
      // result: 漢字
      
      return 0;
    }
    ```
  </Tab>
  <Tab title="CLI">
    ```bash
    # Convert file from Simplified to Traditional
    opencc -i input.txt -o output.txt -c s2t
    
    # Convert stdin
    echo "汉字" | opencc -c s2t
    # Output: 漢字
    ```
  </Tab>
</Tabs>

## Conversion Configurations

OpenCC provides 14 preset configuration files for different conversion scenarios:

<AccordionGroup>
  <Accordion title="Simplified ↔ Traditional">
    - `s2t.json` - Simplified Chinese to Traditional Chinese
    - `t2s.json` - Traditional Chinese to Simplified Chinese
  </Accordion>
  <Accordion title="Taiwan Variants">
    - `s2tw.json` - Simplified to Traditional (Taiwan Standard)
    - `tw2s.json` - Traditional (Taiwan Standard) to Simplified
    - `s2twp.json` - Simplified to Traditional (Taiwan) with Taiwanese idioms
    - `tw2sp.json` - Traditional (Taiwan) to Simplified with Mainland idioms
  </Accordion>
  <Accordion title="Hong Kong Variants">
    - `s2hk.json` - Simplified to Traditional (Hong Kong variant)
    - `hk2s.json` - Traditional (Hong Kong) to Simplified
    - `hk2t.json` - Hong Kong variant to Traditional (OpenCC Standard)
    - `t2hk.json` - Traditional to Hong Kong variant
  </Accordion>
  <Accordion title="Japanese Kanji">
    - `t2jp.json` - Traditional Characters (Kyūjitai) to New Japanese Kanji (Shinjitai)
    - `jp2t.json` - Japanese Kanji (Shinjitai) to Traditional Characters (Kyūjitai)
  </Accordion>
  <Accordion title="Standard Conversions">
    - `t2tw.json` - Traditional (OpenCC Standard) to Taiwan Standard
    - `tw2t.json` - Taiwan Standard to Traditional (OpenCC Standard)
  </Accordion>
</AccordionGroup>

## Trusted By

OpenCC is used in production by major projects:

- **RIME IME** - Cross-platform input method engine
- **ibus-pinyin** - Intelligent Pinyin input method
- **fcitx** - Flexible Input Method Framework
- **GoldenDict** - Feature-rich dictionary lookup program
- **China Biographical Database** - Harvard University research project

<Note>
OpenCC provides conversion between different Chinese character sets and regional variants. It is not a translation tool between Mandarin and Cantonese or other Chinese dialects.
</Note>
