OpenCC uses JSON configuration files to define conversion behavior, specify dictionary loading order, and configure text segmentation. Understanding the configuration system is essential for customizing conversions and creating regional variants.

## Configuration File Structure

OpenCC configuration files are JSON documents with three main components:

```json
{
  "name": "Simplified Chinese to Traditional Chinese",
  "segmentation": {
    "type": "mmseg",
    "dict": {
      "type": "ocd2",
      "file": "STPhrases.ocd2"
    }
  },
  "conversion_chain": [{
    "dict": {
      "type": "group",
      "dicts": [{
        "type": "ocd2",
        "file": "STPhrases.ocd2"
      }, {
        "type": "ocd2",
        "file": "STCharacters.ocd2"
      }]
    }
  }]
}
```

### Configuration Components

<AccordionGroup>
  <Accordion title="name">
    Human-readable description of the conversion configuration. This field is informational and doesn't affect conversion behavior.
  </Accordion>

  <Accordion title="segmentation">
    Defines how input text is split into segments for conversion. The segmentation dictionary is used for Maximum Matching (MaxMatch) word segmentation.
    
    **Fields:**
    - `type`: Segmentation algorithm (currently only `"mmseg"` is supported)
    - `dict`: Dictionary used for segmentation (typically phrase dictionaries)
  </Accordion>

  <Accordion title="conversion_chain">
    An ordered array of conversion steps. Each step applies dictionary-based transformations sequentially. The order matters significantly for multi-stage conversions.
  </Accordion>
</AccordionGroup>

## Conversion Chains

The `conversion_chain` array defines sequential transformation steps. Each step is applied in order, with the output of one step becoming the input of the next.

### Multi-Stage Conversion

Complex conversions often require multiple stages:

<Steps>
  <Step title="Base Conversion">
    Convert between Simplified and Traditional using core dictionaries (phrases first, then characters).
  </Step>
  
  <Step title="Regional Variants">
    Apply regional character variants (e.g., Taiwan or Hong Kong specific forms).
  </Step>
  
  <Step title="Regional Phrases">
    Replace phrases with regional idioms and terminology.
  </Step>
</Steps>

**Example: `s2twp.json` conversion chain**

```json
"conversion_chain": [
  {
    "dict": {
      "type": "group",
      "dicts": [
        {"type": "ocd2", "file": "STPhrases.ocd2"},
        {"type": "ocd2", "file": "STCharacters.ocd2"}
      ]
    }
  },
  {"dict": {"type": "ocd2", "file": "TWPhrases.ocd2"}},
  {"dict": {"type": "ocd2", "file": "TWVariants.ocd2"}}
]
```

This three-stage chain:
1. Converts Simplified to Traditional (phrases prioritized over characters)
2. Replaces Mainland phrases with Taiwan equivalents (鼠標 → 滑鼠)
3. Applies Taiwan character variants (裏 → 裡)

## Dictionary Types

OpenCC supports multiple dictionary formats within configurations:

### Single Dictionary

```json
{
  "dict": {
    "type": "ocd2",
    "file": "STCharacters.ocd2"
  }
}
```

### Dictionary Group

A `group` type allows multiple dictionaries to be searched sequentially. The first match wins:

```json
{
  "dict": {
    "type": "group",
    "dicts": [
      {"type": "ocd2", "file": "STPhrases.ocd2"},
      {"type": "ocd2", "file": "STCharacters.ocd2"}
    ]
  }
}
```

<Warning>
**Dictionary order matters!** Always place phrase dictionaries before character dictionaries to ensure multi-character terms are matched before individual characters.
</Warning>

### Supported Dictionary Formats

| Format | Type Value | Description | Use Case |
|--------|-----------|-------------|----------|
| Text | `text` | Plain text tab-separated | Development, editing |
| MARISA Trie | `ocd2` | Optimized binary format | Production (default) |
| Darts Trie | `ocd` | Legacy binary format | Legacy support |

## Segmentation Configuration

The `segmentation` section configures Maximum Matching (MM) word segmentation:

```json
"segmentation": {
  "type": "mmseg",
  "dict": {
    "type": "ocd2",
    "file": "STPhrases.ocd2"
  }
}
```

### Segmentation Algorithm

**mmseg** (Maximum Matching Segmentation):
- Performs longest prefix matching using the segmentation dictionary
- Splits input into segments before conversion
- Ensures multi-character words are kept together
- Unmatched UTF-8 sequences are preserved by character length

<Note>
The segmentation dictionary is typically a phrase dictionary (e.g., `STPhrases.ocd2`) to ensure compound words and idioms are recognized as single units.
</Note>

## Creating Custom Configurations

### Basic Custom Configuration

```json custom-s2t.json
{
  "name": "Custom Simplified to Traditional",
  "segmentation": {
    "type": "mmseg",
    "dict": {
      "type": "text",
      "file": "custom-phrases.txt"
    }
  },
  "conversion_chain": [{
    "dict": {
      "type": "group",
      "dicts": [
        {"type": "text", "file": "custom-phrases.txt"},
        {"type": "ocd2", "file": "STCharacters.ocd2"}
      ]
    }
  }]
}
```

### Domain-Specific Configuration

Create specialized conversions for technical domains:

```json tech-s2tw.json
{
  "name": "Technical Simplified to Taiwan Traditional",
  "segmentation": {
    "type": "mmseg",
    "dict": {"type": "ocd2", "file": "STPhrases.ocd2"}
  },
  "conversion_chain": [
    {
      "dict": {
        "type": "group",
        "dicts": [
          {"type": "text", "file": "tech-terms.txt"},
          {"type": "ocd2", "file": "STPhrases.ocd2"},
          {"type": "ocd2", "file": "STCharacters.ocd2"}
        ]
      }
    },
    {"dict": {"type": "ocd2", "file": "TWVariants.ocd2"}}
  ]
}
```

Place your custom dictionary first in the group to override default conversions with domain-specific terms.

## Configuration Loading

### Default Configuration Path

OpenCC searches for configuration files in:

1. Current working directory
2. `PKGDATADIR` (typically `/usr/share/opencc/`)
3. Executable directory

### OPENCC_DATA_DIR Environment Variable

Override the default configuration directory:

```bash
export OPENCC_DATA_DIR=/path/to/custom/configs
opencc -c s2t.json -i input.txt -o output.txt
```

<CodeGroup>
```bash Linux/macOS
export OPENCC_DATA_DIR=/opt/opencc/data
opencc -c custom-s2t.json -i input.txt -o output.txt
```

```powershell Windows
$env:OPENCC_DATA_DIR="C:\opencc\data"
opencc -c custom-s2t.json -i input.txt -o output.txt
```

```python Python
import os
import opencc

os.environ['OPENCC_DATA_DIR'] = '/path/to/configs'
converter = opencc.OpenCC('custom-s2t.json')
```

```javascript Node.js
process.env.OPENCC_DATA_DIR = '/path/to/configs';
const { OpenCC } = require('opencc');
const converter = new OpenCC('custom-s2t.json');
```
</CodeGroup>

### Absolute Configuration Paths

You can specify absolute paths to configuration files:

```cpp
opencc::SimpleConverter converter("/path/to/custom-s2t.json");
```

```python
import opencc
converter = opencc.OpenCC('/path/to/custom-s2t.json')
```

## Configuration Best Practices

<AccordionGroup>
  <Accordion title="Always prioritize phrases over characters">
    In dictionary groups, place phrase dictionaries before character dictionaries to ensure compound words match correctly:
    
    ```json
    "dicts": [
      {"type": "ocd2", "file": "STPhrases.ocd2"},  // First
      {"type": "ocd2", "file": "STCharacters.ocd2"}  // Second
    ]
    ```
  </Accordion>

  <Accordion title="Use text format for development">
    During development, use `text` dictionaries for easy editing. Convert to `ocd2` for production:
    
    ```bash
    opencc_dict -i custom.txt -o custom.ocd2 -f text -t ocd2
    ```
  </Accordion>

  <Accordion title="Chain conversions carefully">
    The order of conversion chain steps matters significantly. Test multi-stage conversions thoroughly:
    
    - Base conversion first
    - Regional phrases second
    - Regional variants last
  </Accordion>

  <Accordion title="Keep configurations modular">
    Create separate configurations for different conversion scenarios rather than one large catch-all configuration.
  </Accordion>
</AccordionGroup>

## Troubleshooting

### Configuration Not Found

**Error:** `Cannot open configuration file`

**Solutions:**
- Verify the configuration file exists in the search path
- Use an absolute path to the configuration file
- Set `OPENCC_DATA_DIR` to the correct directory
- Check file permissions

### Dictionary File Not Found

**Error:** `Dictionary file not found`

**Solutions:**
- Ensure all referenced dictionary files exist
- Use absolute paths in the configuration
- Verify dictionary files are in the same directory as the configuration
- Check that binary dictionaries (`.ocd2`) are properly compiled

### Incorrect Conversion Results

**Symptoms:** Wrong characters or missing conversions

**Solutions:**
- Verify dictionary order in groups (phrases before characters)
- Check conversion chain order
- Inspect dictionaries for missing or incorrect entries
- Test with simpler configurations to isolate issues
