This guide will help you perform your first Chinese text conversion using OpenCC. We'll cover the basics and get you up and running quickly.

## Your First Conversion

Let's start with a simple Simplified to Traditional Chinese conversion.

<Tabs>
  <Tab title="Command Line">
    Convert text using the `opencc` command:

    ```bash
    echo "汉字" | opencc -c s2t
    ```

    Output:
    ```
    漢字
    ```

    Convert a file:
    ```bash
    opencc -c s2t -i input.txt -o output.txt
    ```
  </Tab>
  <Tab title="Python">
    ```python
    import opencc

    # Create a converter
    converter = opencc.OpenCC('s2t')

    # Convert text
    result = converter.convert('汉字')
    print(result)  # Output: 漢字
    ```
  </Tab>
  <Tab title="Node.js">
    ```javascript
    const OpenCC = require('opencc');

    // Create a converter
    const converter = new OpenCC('s2t.json');

    // Convert text synchronously
    const result = converter.convertSync('汉字');
    console.log(result);  // Output: 漢字
    ```
  </Tab>
  <Tab title="C++">
    ```cpp
    #include "opencc.h"
    #include <iostream>

    int main() {
      // Create a converter
      const opencc::SimpleConverter converter("s2t.json");

      // Convert text
      std::string result = converter.Convert("汉字");
      std::cout << result << std::endl;  // Output: 漢字

      return 0;
    }
    ```

    Compile with:
    ```bash
    g++ -std=c++17 example.cpp -lopencc -o example
    ```
  </Tab>
</Tabs>

<Tip>
**Configuration naming convention:**
- Python and CLI: Use short names like `s2t` (automatically appends `.json`)
- Node.js and C++: Use full names like `s2t.json`
</Tip>

## Step-by-Step Tutorial

Let's build a more complete example that demonstrates common conversion scenarios.

<Steps>
  <Step title="Choose your conversion type">
    OpenCC supports many conversion types. The most common are:

    - `s2t` - Simplified → Traditional Chinese (standard)
    - `t2s` - Traditional → Simplified Chinese
    - `s2tw` - Simplified → Taiwan Traditional
    - `s2hk` - Simplified → Hong Kong Traditional
    - `s2twp` - Simplified → Taiwan Traditional with idioms

    For this tutorial, we'll use `s2twp` to see regional vocabulary differences.
  </Step>

  <Step title="Create the converter">
    <Tabs>
      <Tab title="Python">
        ```python
        import opencc

        # Create converter for Simplified to Taiwan Traditional with idioms
        converter = opencc.OpenCC('s2twp')
        ```
      </Tab>
      <Tab title="Node.js">
        ```javascript
        const OpenCC = require('opencc');

        // Create converter for Simplified to Taiwan Traditional with idioms
        const converter = new OpenCC('s2twp.json');
        ```
      </Tab>
      <Tab title="C++">
        ```cpp
        #include "opencc.h"

        int main() {
          // Create converter for Simplified to Taiwan Traditional with idioms
          const opencc::SimpleConverter converter("s2twp.json");
        ```
      </Tab>
    </Tabs>
  </Step>

  <Step title="Convert text with regional vocabulary">
    Notice how OpenCC converts not just characters, but also regional vocabulary:

    <Tabs>
      <Tab title="Python">
        ```python
        # Example with regional vocabulary
        mainland_text = "鼠标和软件都是计算机的重要组成部分。"
        taiwan_text = converter.convert(mainland_text)

        print("Mainland:", mainland_text)
        print("Taiwan:  ", taiwan_text)
        ```

        Output:

        Taiwan:   滑鼠和軟體都是電腦的重要組成部分。

        Key differences:
        - 鼠标 → 滑鼠 (mouse)
        - 软件 → 軟體 (software)
        - 计算机 → 電腦 (computer)
      </Tab>

          console.log("Taiwan:  ", taiwanText);
        }

        Output:

        Mainland: 鼠标和软件都是计算机的重要组成部分。
        Taiwan:   滑鼠和軟體都是電腦的重要組成部分。

## Understanding Configuration Files

Configuration files define how OpenCC performs conversions. They specify:

1. **Segmentation** - How to break text into words
2. **Dictionaries** - Which conversion dictionaries to use
3. **Conversion chain** - The order of dictionary application

Example configuration (`s2t.json`):

```json
{
  "name": "Simplified Chinese to Traditional Chinese",
  "segmentation": {
    "type": "mmseg",
    "dict": {
      "type": "ocd2",
      "file": "STPhrases.ocd2"
    }
  },
  "conversion_chain": [{
    "dict": {
      "type": "group",
      "dicts": [
        {"type": "ocd2", "file": "STPhrases.ocd2"},
        {"type": "ocd2", "file": "STCharacters.ocd2"}
      ]
    }
  }]
}
```

<Info>
The configuration uses phrase-level dictionaries first (`STPhrases`), then falls back to character-level conversion (`STCharacters`). This ensures accurate conversion of multi-character words.
</Info>

## Command-Line Usage

The `opencc` command provides powerful text conversion capabilities:

### Basic Usage

```bash
# Convert from stdin to stdout
echo "汉字" | opencc -c s2t

# Convert a file
opencc -c s2t -i input.txt -o output.txt

# Convert with specific configuration
opencc -c s2twp -i mainland.txt -o taiwan.txt
```

### Options

```bash
opencc --help
```

| Option | Description |
|--------|-------------|
| `-c, --config <file>` | Configuration file (e.g., `s2t`, `t2s`, `s2tw`) |
| `-i, --input <file>` | Input file (default: stdin) |
| `-o, --output <file>` | Output file (default: stdout) |

### Advanced Examples

<CodeGroup>
```bash Pipe multiple commands
# Convert Simplified to Traditional, then to Taiwan variant
echo "计算机软件" | opencc -c s2t | opencc -c t2tw
```

```bash Batch file conversion
# Convert all .txt files in a directory
for file in *.txt; do
  opencc -c s2tw -i "$file" -o "tw_${file}"
done
```

```bash Process large files
# Convert large file with progress indicator
pv large_file.txt | opencc -c s2t > output.txt
```
</CodeGroup>

## Node.js API Variations

Node.js offers three API styles:

<CodeGroup>
```javascript Synchronous (blocking)
const OpenCC = require('opencc');
const converter = new OpenCC('s2t.json');

const result = converter.convertSync('汉字');
console.log(result);
```

```javascript Callback-based
const OpenCC = require('opencc');
const converter = new OpenCC('s2t.json');

converter.convert('汉字', (err, converted) => {
  if (err) {
    console.error(err);
    return;
  }
  console.log(converted);
});
```

```javascript Promise-based
const OpenCC = require('opencc');
const converter = new OpenCC('s2t.json');

converter.convertPromise('汉字')
  .then(converted => console.log(converted))
  .catch(err => console.error(err));
```

```javascript Async/Await
const OpenCC = require('opencc');

async function main() {
  const converter = new OpenCC('s2t.json');
  const result = await converter.convertPromise('汉字');
  console.log(result);
}

main();
```

```typescript TypeScript
import { OpenCC } from 'opencc';

async function main(): Promise<void> {
  const converter: OpenCC = new OpenCC('s2t.json');
  const result: string = await converter.convertPromise('汉字');
  console.log(result);
}

main();
```
</CodeGroup>

## Common Conversion Scenarios

### Website Localization

Convert web content for different Chinese-speaking regions:

```python
import opencc

# Create converters for different regions
s2tw = opencc.OpenCC('s2twp')  # Mainland → Taiwan
s2hk = opencc.OpenCC('s2hk')   # Mainland → Hong Kong

mainland_content = "软件下载：点击这里下载最新版本。"

taiwan_content = s2tw.convert(mainland_content)
hk_content = s2hk.convert(mainland_content)

print("Taiwan:", taiwan_content)   # 軟體下載：點選這裡下載最新版本。
print("Hong Kong:", hk_content)    # 軟件下載：點擊這裏下載最新版本。
```

### Document Processing

Convert entire documents:

```python
import opencc

converter = opencc.OpenCC('s2t')

# Read and convert file
with open('input_simplified.txt', 'r', encoding='utf-8') as f:
    content = f.read()

converted = converter.convert(content)

# Write converted content
with open('output_traditional.txt', 'w', encoding='utf-8') as f:
    f.write(converted)
```

### API Response Translation

Integrate with web APIs:

```javascript
const OpenCC = require('opencc');
const express = require('express');

const app = express();
const converter = new OpenCC('s2t.json');

app.get('/convert', async (req, res) => {
  const text = req.query.text;
  const converted = await converter.convertPromise(text);
  res.json({ original: text, converted });
});

app.listen(3000);
```

## Best Practices

<CardGroup cols={2}>
  <Card title="Reuse Converter Instances" icon="recycle">
    Creating a converter loads dictionaries into memory. Reuse the same instance for multiple conversions.
  </Card>
  <Card title="Choose the Right Configuration" icon="crosshairs">
    Use specific configurations like `s2twp` for regional idioms, not just character conversion.
  </Card>
  <Card title="Handle UTF-8 Properly" icon="language">
    Always use UTF-8 encoding for input and output files to avoid character corruption.
  </Card>
  <Card title="Use Async APIs in Node.js" icon="rocket">
    Prefer `convertPromise()` over `convertSync()` to avoid blocking the event loop.
  </Card>
</CardGroup>
